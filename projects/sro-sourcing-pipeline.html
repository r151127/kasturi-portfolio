Impact

Quantitative

  • 346 → 157 → 90 candidates: clean funnel with zero manual triage
  • 100% phone reveal success across 90 candidates
  • 89% resume retrieval (80/90) in first run
  • Credits consumed only on the final filtered pool, not the raw 346

Qualitative

  • Unqualified profiles rejected at database level before human review
  • 6 regional managers with independent, language-mapped queues
  • Self-serve pool sizing before committing reviewer time
  • Policy changes (tightening exp band) = one DB operation, not a rebuild
Key Learning

Build on the existing data model, not beside it

Building the automation directly on top of Naukri's existing data model — rather than introducing a parallel system — meant retroactive policy changes like tightening the experience band became a single reversible database operation instead of a rebuild. Baking hard filters (CTC cap, experience band) into the pipeline itself, not just the UI, kept the candidate pool clean by construction rather than by manual review discipline.
